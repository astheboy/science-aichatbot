# 교사가 설계하는 AI 수업
지금까지 우리는 AI 튜터의 '뇌(Brain)', '눈(Eyes)', '메타인지(Metacognition)'를 만들었습니다. 하지만 아무리 뛰어난 AI라도, 교실 현장의 맥락(Context)을 이해하지 못하면 무용지물입니다.

AI 튜터가 어떤 역할을 맡아야 할지, 이번 수업의 학습 목표는 무엇인지, 학생들의 수준은 어떤지... 이 모든 **수업 디자인(Instructional Design)** 의 주체는 결국 '교사'여야 합니다.

이번 에피소드에서는 교사가 AI 튜터를 자신의 수업 의도대로 커스터마이징하고, 학생들의 학습 과정을 모니터링할 수 있는 **교사용 대시보드(Teacher Dashboard)** 의 개발 과정을 다룹니다.

<!-- truncate -->

## 1. 대시보드 기획: 교사를 위한 '관제탑'
교사용 대시보드는 단순히 기능을 켜고 끄는 설정창이 아닙니다. 교사가 AI 튜터와 협업하여 수업을 운영하는 '관제탑' 역할을 해야 합니다.

### 핵심 기능 요구사항
1.  **간편한 인증**: 별도 회원가입 없이 학교 계정(Google)으로 바로 시작.
2.  **수업(Lesson) 관리**: 매 차시별로 다른 주제와 AI 페르소나 설정.
3.  **프롬프트 엔지니어링 도구**: 복잡한 프롬프트를 몰라도 '템플릿'으로 쉽게 AI 설정.
4.  **학습 자료 연동**: 교사가 가진 학습지, 영상 링크를 AI에게 '지식'으로 주입 (RAG).
5.  **학생 분석(Analytics)**: 대화 로그를 통해 학생의 사고 발달 과정을 분석.

## 2. 수업 관리와 AI 페르소나 설정 (`teacher.html`)
가장 먼저 구현한 것은 수업 생성 기능입니다. 교사는 여기서 **"오늘 AI가 어떤 선생님이 될지"** 를 정합니다.

### 직관적인 설정 UI
수업을 만들 때 교사는 다음 항목들을 설정합니다.
*   **과목 & 주제**: 예) "과학 - 식물의 광합성"
*   **학생용 설명**: 학생들의 접속 화면에 뜰 수업 안내.
*   **AI 지시사항(System Instruction)**: 여기가 핵심입니다. AI에게 구체적인 '역할'을 부여합니다.

> "너는 친절한 숲 해설가야. 광합성을 '요리'에 비유해서 설명해줘."

하지만 모든 교사가 프롬프트를 잘 짤 수는 없습니다. 그래서 **'사고력 증진 템플릿'** 시스템을 도입했습니다.

### 🧩 사고력 증진 템플릿 (Thinking Modes)
`teacher.html` 코드를 보면, 사전에 정의된 강력한 프롬프트 템플릿들이 있습니다. 교사는 버튼 클릭 한 번으로 고도화된 교수법을 AI에 장착시킬 수 있습니다.

*   **🔍 탐구 모드 (Inquiry)**: 정답 대신 질문을 던져 학생 스스로 가설을 세우게 함. (과학/사회)
*   **💬 토론 모드 (Debate)**: AI가 중립적 사회자가 되어 반대 의견을 제시하며 사고 확장. (사회/국어)
*   **🧩 문제해결 모드 (Problem Solving)**: 폴리아(Polya)의 문제해결 4단계에 따라 힌트 제공. (수학)
*   **🎭 역할극 모드 (Role Playing)**: 역사적 인물이 되어 그 시대의 관점에서 대화. (역사)
*   **🧑‍🏫 거꾸로 학습 모드 (Flipped Learning)**: AI가 학생이 되고, 진짜 학생이 AI를 가르치며 학습.

```javascript
/* public/teacher.html 내 템플릿 예시 */
const THINKING_MODES = {
    flipped: {
        name: '🧑‍🏫 거꾸로 학습 모드',
        template: `**역할**: 너는 오늘 배울 내용에 대해 아무런 사전 지식이 없는, 총명하고 호기심 많은 학생이야.
        **목표**: 학생이 선생님이 되어 설명하는 것을 듣고, 질문을 통해 네가 완벽히 이해하는 척 연기해.
        **규칙**:
        1. 학생보다 먼저 아는 척 하지 마.
        2. 학생의 설명이 논리적으로 비약되면 "그건 왜 그런 거야?"라고 순진하게 물어봐.`
    }
    // ...
};
```

이 템플릿 덕분에 교사는 프롬프트 엔지니어링을 몰라도 **"오늘은 거꾸로 학습으로 해볼까?"** 하며 다양한 수업 모형을 시도할 수 있습니다.

## 3. 학습 자료 주입: RAG 파이프라인 연동
Ep.4에서 다룬 **RAG(검색 증강 생성)** 기술이 여기서 빛을 발합니다. 교사는 대시보드에서 수업 자료를 업로드합니다.

*   **웹 링크**: 유튜브 영상, 뉴스 기사 URL
*   **파일 업로드**: 수업용 PDF, 학습지 이미지 (OCR로 텍스트 추출)

이 자료들은 즉시 벡터 데이터베이스(화면 뒤에서는 Firestore와 임베딩)에 저장되고, 학생이 질문할 때 AI가 이 자료를 참고해서 답변하게 됩니다. 즉, **"교과서 밖의 이야기는 하지 마"** 또는 **"이 학습지의 내용 중심으로 설명해"** 가 가능해지는 것입니다.

## 4. 학생 분석: 단순 점수가 아닌 '사고력 측정'
AI 튜터의 가장 큰 장점은 모든 대화가 데이터로 남는다는 것입니다. 하지만 교사가 수십 명의 대화 로그를 일일이 다 읽을 수는 없습니다.
그래서 **'AI가 분석하는 AI 수업'** 기능을 서버(`functions/index.js`)에 구현했습니다.

### `analyzeStudentConversations` 함수 분석
이 함수는 학생의 대화 기록 전체를 Gemini Pro에게 던져주고, 교육학적 관점에서 분석 보고서를 써달라고 요청합니다.

**분석 프롬프트의 핵심 (Bloom's Taxonomy 기반):**
단순히 "참가 잘 함" 수준이 아니라, 구체적인 발달 지표를 측정합니다.

1.  **발달 단계 평가**: 기억 → 이해 → 적용 → 분석 → 종합 → 평가 (Bloom의 분류학)
2.  **형식적 조작기 전환**: 구체적 경험에 의존하는지, 추상적/가설적 사고가 가능한지 분석.
3.  **학습 태도**: 호기심, 끈기, 메타인지 활용 능력.

```javascript
/* functions/index.js (요약) */
const giftedAnalysisPrompt = `
학생의 AI 튜터 대화를 분석하여 성장 가능성과 발달 단계를 평가해주세요.
1. 현재 발달 단계 (Bloom의 교육목표 분류학 기반)
2. 사고 발달 지표 (구체적 조작기 -> 형식적 조작기 전환 정도)
3. 성장 예측 (1-2개월 단기, 학기 단위 중기 목표)
4. 맞춤형 지도 전략 제안
...
`;
```

교사는 이 분석 결과를 통해 "A학생은 개념 이해는 완벽하지만, 이를 새로운 상황에 적용하는 데 어려움을 겪고 있구나. 다음 시간에는 응용 문제를 더 줘봐야겠다."라는 구체적인 **데이터 기반 피드백**을 얻을 수 있습니다.

## 5. 마치며: 기술을 넘어 교육으로
이 교사용 대시보드를 만들면서 가장 신경 쓴 부분은 **'기술적 화려함'보다 '교육적 실용성'** 이었습니다.
*   복잡한 AI 모델 설정보다는 **'수업 템플릿'** 제공
*   단순 로그 나열보다는 **'교육학적 분석 리포트'** 제공

이로써 우리 챗봇 시스템은 학생에게는 '맞춤형 튜터', 교사에게는 '든든한 보조교사'가 되었습니다.

이제 시스템의 핵심 기능들은 모두 소개했습니다. 다음 마지막 에피소드에서는 이 프로젝트의 **로드맵**과 **개발 회고(Retrospective)** 로 시리즈를 마무리하겠습니다.
