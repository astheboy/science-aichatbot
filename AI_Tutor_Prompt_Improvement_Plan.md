# AI 튜터 챗봇 프롬프트 개선 계획서

## 1. 연구 분석 요약

### 핵심 연구 결과
ChatGPT vs 인간 전문가 연구에서 발견된 주요 문제점:

1. **메타인지 오프로딩**: 학생들이 스스로 문제를 진단하는 '질문 진단' 단계를 생략하고 AI에게 의존
2. **실행적 도움 탐색 과다**: 직접적인 답을 요구하는 경향이 높음
3. **도움 평가 단계 생략**: AI 응답에 대한 비판적 평가 없이 수용
4. **성찰 부족**: 이전 도움을 다시 검토하는 성찰적 학습 행동 부족

### 교육적 시사점
- AI 챗봇은 단순한 '정답 자판기'가 아닌 **'학습 파트너'**로 설계되어야 함
- **메타인지 스캐폴딩**을 통해 학습자의 자기 조절 능력 촉진 필요
- **도구적 도움 탐색**을 장려하여 깊이 있는 학습 유도
- **성찰적 학습**을 통한 통합적 사고 능력 개발

## 2. 현재 시스템 분석

### 현재 프롬프트 시스템의 강점
1. **적응형 프롬프트**: ResponseAnalyzer가 학생 응답을 6가지 유형으로 분류
2. **맥락 인식**: 대화 이력을 고려한 프롬프트 생성
3. **과목별 특화**: JSON 기반 과목별 맞춤 프롬프트
4. **교사 커스터마이징**: 교사별 맞춤 프롬프트 지원

### 현재 시스템의 한계점
1. **메타인지 지원 부족**: 학생의 자기 진단 유도 기능 미흡
2. **실행적 질문 대응**: 직접적 답변 요구에 대한 리프레이밍 전략 부족
3. **평가 촉진 미흡**: AI 응답에 대한 학생의 평가 유도 부족
4. **성찰 지원 제한**: 이전 대화 참조 및 요약 기능 부족

## 3. 개선 방향 및 구체적 계획

### 3.1 메타인지 스캐폴딩 강화

#### 현재 문제점
```javascript
// 현재 시스템: 학생이 "내 실험이 왜 실패했어?"라고 물으면 바로 분석 제공
// 문제: 학생의 자기 진단 기회 상실
```

#### 개선안
```javascript
// 개선된 시스템: 질문 진단 단계 유도
const metacognitiveScaffolding = {
    patterns: [
        /왜.*실패/,
        /뭐가.*문제/,
        /어떻게.*해야/
    ],
    response_strategy: "diagnosis_first",
    scaffolding_questions: [
        "좋은 질문이에요! 실험 과정을 차근차근 생각해보면서, 어느 단계에서 예상과 다른 결과가 나왔는지 먼저 찾아볼까요?",
        "실험을 다시 떠올려보세요. 1) 준비 단계, 2) 실행 단계, 3) 관찰 단계 중 어디서 가장 의심스러운 부분이 있었나요?"
    ]
}
```

### 3.2 도구적 도움 탐색 장려

#### 현재 문제점
```javascript
// 현재: 학생이 "답을 알려줘"라고 하면 소크라테스식 질문만 제공
// 문제: 학생 좌절감 증가, 실행적 도움 탐색 지속
```

#### 개선안
```javascript
const instrumentalHelpStrategy = {
    executive_request_patterns: [
        /답.*알려/,
        /정답.*뭐/,
        /어떻게.*하면.*돼/
    ],
    reframing_approach: {
        acknowledgment: "정답을 바로 알고 싶은 마음 이해해요!",
        choice_offering: "두 가지 방법이 있어요. 1) 함께 단계별로 찾아보기 2) 힌트부터 받아보기 중 어느 것이 좋을까요?",
        scaffolding_path: "그럼 먼저 작은 힌트를 드릴까요? 이 실험에서 가장 중요한 요소는 무엇이라고 생각하세요?"
    }
}
```

## 4. 구현 우선순위

### Phase 1: 메타인지 스캐폴딩 (2주)
- ResponseAnalyzer에 실행적 질문 패턴 탐지 추가
- PromptBuilder에 질문 진단 유도 로직 구현
- 기존 프롬프트에 메타인지 스캐폴딩 요소 추가

### Phase 2: 도구적 도움 탐색 장려 (2주)
- 질문 리프레이밍 시스템 구현
- 단계별 힌트 제공 메커니즘 개발
- 학습자 선택권 제공 인터페이스 구축

### Phase 3: 평가 및 성찰 시스템 (3주)
- 응답 후 평가 유도 시스템 구현
- 대화 이력 참조 및 요약 기능 개발
- 성찰적 학습 지원 도구 구축

### Phase 4: 통합 테스트 및 최적화 (1주)
- 전체 시스템 통합 테스트
- 교사 피드백 수집 및 반영
- 성능 최적화

## 5. 예상 효과

1. **메타인지 능력 향상**: 학생들이 스스로 문제를 진단하고 학습 과정을 모니터링
2. **깊이 있는 학습**: 직접적 답변보다 탐구 과정을 통한 깊이 있는 이해
3. **비판적 사고**: AI 응답에 대한 평가를 통한 비판적 사고력 발달
4. **자기조절학습**: 성찰을 통한 자기주도적 학습 능력 강화

## 6. 성공 지표

- 학생의 자발적 질문 진단 행동 증가 (현재 대비 40% 증가 목표)
- 도구적 질문 비율 증가 (실행적:도구적 = 6:4 → 4:6)
- 응답 평가 참여율 (80% 이상)
- 이전 대화 참조 빈도 증가 (세션당 평균 2회 이상)
